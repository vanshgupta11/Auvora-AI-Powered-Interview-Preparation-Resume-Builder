# Auvora — Deployment Guide

This app is deployed in two parts:
| Part | Platform | Why |
|------|----------|-----|
| **Frontend** (React + Vite) | [Vercel](https://vercel.com) | Best-in-class static + SPA hosting |
| **Backend** (Express + Puppeteer) | [Render](https://render.com) | Docker support required for Chromium/PDF generation |

---

## Step 1 — Deploy the Backend on Render

> **Do this first** so you have the backend URL to paste into Vercel.

1. Go to [https://render.com](https://render.com) and sign in with GitHub.
2. Click **New → Web Service**.
3. Connect to your GitHub repo: `vanshgupta11/Auvora-AI-Powered-Interview-Preparation-Resume-Builder`
4. Configure the service:
   - **Name**: `auvora-backend`
   - **Root Directory**: leave blank (Render uses `render.yaml` at root)
   - **Runtime**: Docker
   - **Dockerfile Path**: `./Backend/Dockerfile`
   - **Docker Context**: `./Backend`
5. Alternatively, click **New → Blueprint** and Render will auto-read `render.yaml`.

### Environment Variables to set in Render Dashboard:
| Key | Value |
|-----|-------|
| `MONGO_URI` | Your MongoDB Atlas connection string |
| `GOOGLE_GENAI_API_KEY` | Your Google Gemini API key |
| `FRONTEND_URL` | Your Vercel frontend URL (set this after Step 2) |
| `JWT_SECRET` | Auto-generated by Render (or set manually) |
| `NODE_ENV` | `production` |

> After deploy, your backend URL will be: `https://auvora-backend.onrender.com`  
> (or a custom name you chose — copy it for Step 2)

---

## Step 2 — Deploy the Frontend on Vercel

1. Go to [https://vercel.com](https://vercel.com) and sign in with GitHub.
2. Click **Add New → Project**.
3. Import your GitHub repo: `vanshgupta11/Auvora-AI-Powered-Interview-Preparation-Resume-Builder`
4. Configure the project:
   - **Root Directory**: `Frontend`
   - **Framework Preset**: Vite (auto-detected)
   - **Build Command**: `npm run build`
   - **Output Directory**: `dist`
5. Add **Environment Variable**:
   | Key | Value |
   |-----|-------|
   | `VITE_API_URL` | `https://auvora-backend.onrender.com` (your Render URL from Step 1) |
6. Click **Deploy**.

> After deploy, your frontend URL will be: `https://auvora-<something>.vercel.app`

---

## Step 3 — Connect Frontend URL back to Backend

1. Go to your Render dashboard → `auvora-backend` service → **Environment**.
2. Update `FRONTEND_URL` to your Vercel URL (e.g. `https://auvora.vercel.app`).
3. Click **Save Changes** — Render will auto-redeploy.

This updates the CORS allowed origin on the backend so your Vercel frontend can call the API.

---

## Step 4 — Verify

- Open your Vercel URL in the browser.
- Register/login → should work (auth calls the Render backend).
- Upload a resume and generate an interview report.
- Download a PDF resume.

## Local Development

```bash
# Backend
cd Backend
cp .env.example .env   # fill in your actual values
npm install
npm run dev            # runs on http://localhost:3000

# Frontend (in a separate terminal)
cd Frontend
echo "VITE_API_URL=http://localhost:3000" > .env.local
npm install
npm run dev            # runs on http://localhost:5173
```

## Notes

- **Render free tier** spins down after 15 minutes of inactivity. The first request after spin-down may take ~30 seconds.
- **Puppeteer** (PDF generation) requires Chromium — this is why Docker is used on Render.
- **MongoDB Atlas** free tier (M0) works fine for this project.
